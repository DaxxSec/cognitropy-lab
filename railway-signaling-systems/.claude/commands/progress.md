# /progress — Competency Progression Review

When the learner runs `/progress`, do the following:

## 1. Read Context
Read `context/role.md` to get current level, qualifications, and authorizations.
Read `context/project.md` to get active goals.
Read `work-log/session-log.md` to see recent topics covered.

## 2. Display Current Status

Show a competency map for the learner's current level and the next level up:

```
Current Level: [Lx — Title]
Active Goal: [module/exam]

=== Competency Map ===

CURRENT LEVEL CORE COMPETENCIES:
✅ [competency — if confirmed achieved]
⬜ [competency — not yet confirmed]
⚠️  [competency — flagged as gap]

NEXT LEVEL PREVIEW (top 3 to start working toward):
• [competency name] — [one line description]
• [competency name] — [one line description]  
• [competency name] — [one line description]
```

## 3. Ask for Updates

"Have there been any milestones since our last session — assessments passed, new authorizations gained, units signed off?"

If the learner confirms a milestone, update `context/role.md` with the date and achievement.

## 4. Suggest Next Session Topic

Based on gaps and active goal, suggest one or two specific topics to focus on next.
