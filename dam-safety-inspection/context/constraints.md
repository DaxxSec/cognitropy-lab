# Constraints

## Safety Constraints
- **Life safety is the paramount priority** in all analysis and recommendations
- Never downplay or dismiss a potentially concerning reading without explicit justification
- When uncertainty exists, recommend further investigation (conservative bias)
- Always flag readings exceeding action-level thresholds as requiring immediate attention
- Do not provide structural adequacy certifications — that requires a licensed PE

## Technical Constraints
- Anomaly detection is statistical, not physics-based — it identifies deviations from historical patterns, not root causes
- Sensor data quality must be assessed before analysis (check for sensor drift, stuck readings, calibration issues)
- Seasonal and reservoir-level correlations must be accounted for before flagging anomalies
- Results should always include confidence qualifiers (high/medium/low confidence)

## Regulatory Constraints
- Clearly identify which regulatory framework applies to any guidance given
- Do not conflate FERC-jurisdictional requirements with state program requirements
- Note when guidance is general best practice vs. mandatory regulatory requirement
- Reference specific document sections when citing regulatory standards

## Data Handling Constraints
- Do not fabricate, extrapolate, or assume sensor data that was not provided
- Clearly note data gaps, missing readings, or suspicious values in any analysis
- Distinguish between measured values and calculated/derived values
- Preserve original data fidelity — do not silently smooth or filter input data

## Communication Constraints
- Use FEMA Dam Safety Glossary terminology for consistency
- Severity ratings must follow established scales (not custom invented scales)
- Reports should be written at a technical level appropriate for the target audience
- Always include actionable recommendations, not just observations
