# Dam Safety Inspection Protocols

A Claude Agent workspace for dam safety inspection, structural health monitoring, and automated anomaly detection in dam instrumentation data.

## Overview

This workspace equips a Claude agent with deep knowledge of dam safety engineering — from routine visual inspections to interpreting complex instrumentation arrays. It combines regulatory framework awareness (FEMA, FERC, USACE, ICOLD) with automated anomaly detection techniques to help engineers identify potential failure precursors before they become emergencies.

Dams are among the most consequential pieces of civil infrastructure. There are over 91,000 dams in the US National Inventory of Dams, and approximately 15,600 are classified as high-hazard potential (meaning failure would likely cause loss of life). Regular, thorough inspection is the frontline defense against catastrophic failure, yet the volume of instrumentation data generated by modern monitoring systems often exceeds what inspection teams can manually review.

## Key Features

### Inspection Support
- Generates comprehensive inspection checklists tailored to dam type (embankment, concrete gravity, arch, buttress, RCC)
- Walks through FEMA's standardized visual inspection protocols
- Flags conditions matching known failure mode indicators (piping, seepage, settlement, cracking patterns)
- Produces structured condition assessment reports with severity ratings

### Anomaly Detection
- Analyzes piezometric head data for abnormal pore pressure trends
- Detects statistically significant deviations in seepage flow measurements
- Monitors inclinometer and settlement gauge readings for movement patterns
- Applies threshold-based alerting (action level, alert level, failure level)
- Implements seasonal decomposition to separate normal cyclical behavior from genuine anomalies

### Regulatory Compliance
- References FEMA Federal Guidelines for Dam Safety (FEMA 93, 145, 333, P-1025)
- Aligns with FERC Engineering Guidelines for dam owners under federal jurisdiction
- Supports USACE Engineering Regulation ER 1110-2-1156 safety evaluation procedures
- Cross-references ICOLD bulletins for international best practices

### Emergency Planning
- Reviews Emergency Action Plan (EAP) components for completeness
- Assists with dam breach parameter estimation
- Supports inundation mapping scenario development
- Generates notification flowchart templates for emergency response

## Dam Types Covered

- **Embankment Dams** (earthfill, rockfill) — seepage, piping, slope stability focus
- **Concrete Gravity Dams** — sliding, overturning, uplift pressure analysis
- **Concrete Arch Dams** — abutment stability, thermal stress, joint behavior
- **Buttress Dams** — strut and slab integrity, foundation concerns
- **RCC Dams** — lift joint quality, permeability patterns
- **Timber Crib and Masonry Dams** — deterioration assessment, retrofit evaluation

## Use Cases

1. **Routine Inspection Prep** — Generate a dam-type-specific checklist before a scheduled inspection visit
2. **Instrumentation Data Review** — Feed CSV/time-series data from piezometers and weirs for automated anomaly flagging
3. **Post-Event Assessment** — After seismic events or extreme precipitation, rapidly screen monitoring data for concerning changes
4. **Regulatory Report Drafting** — Produce inspection reports formatted for FERC Part 12 or state regulatory submissions
5. **Training & Education** — Walk junior inspectors through failure mode identification and severity rating protocols

## Getting Started

1. Clone or open this workspace in Claude Code
2. Run `/onboard` to orient the agent to the dam safety domain
3. Use `/inspect` with your dam profile to generate a tailored inspection checklist
4. Feed instrumentation data via `/analyze-readings` for automated anomaly detection
5. Generate reports using the prompts in `prompts/`

## Technical Requirements

- Claude Code or compatible Claude agent environment
- CSV or JSON instrumentation data for analysis features
- No external API dependencies — all analysis uses statistical methods described in domain knowledge
