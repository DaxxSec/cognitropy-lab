# Project Context — Glassblowing Form Shaping

This file is populated through `/onboard`. Until then it captures the canonical structure plus a worked example.

## Project Identity

- **Studio name:** to be set during `/onboard`
- **Primary practice:** hot-shop blowing / lampworking / fusing / mixed (set during onboarding)
- **Workspace started:** date of first `/onboard` run
- **Workspace owner:** the artist responsible for the workspace's records

## Target Form Vocabulary

The set of forms the studio actively works in. Drives `/form-sim`'s defaults and the sag-threshold lookups.

Examples:
- **Vessels** — cups, tumblers, vases, bottles
- **Goblets** — stemware with foot, stem, cup
- **Plates** — flat to slightly bowled forms
- **Sculptural** — non-functional, often asymmetric or extended
- **Lighting components** — pendant shades, sconces
- **Beads** — lampworking-only
- **Paperweights** — solid, decorated interior
- **Cane / murrine** — feedstock for other forms

## Project Goals

What success looks like for this workspace, in the workspace owner's words. Drives prioritization of form sim work.

Examples:
- "Reproduce the spring 2026 goblet line consistently — currently a 30% reject rate"
- "Develop a 350 mm folded-rim plate form that survives anneal"
- "Document the studio's color palette compatibility tests so apprentices can rely on the record"
- "Build a 90-day shelf-check baseline so I can detect supplier drift earlier"

## Near-Term Backlog
