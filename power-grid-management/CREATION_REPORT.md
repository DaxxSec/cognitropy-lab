## Design Notes

- **Agent persona:** Grid Spatial Analyst — a spatial-first grid analyst rather than a mentor tracking apprenticeship levels. This fit the day's technique (geographic/spatial analysis) better than retrofitting the railway template's "Apprenticeship Progression Tracking" frame.
- **Slash commands:** `/onboard`, `/analyze`, `/map`, `/outage`, `/topology` — each with an explicit "what the agent should NOT do" section to tighten the scope away from operational decision-making.
- **Data-sensitivity posture:** CIP-014 / BCSI treated as first-class constraints in `context/constraints.md` and the `/onboard` flow. Reflects the reality that utility spatial data is usually sensitive.
- **Cyber-physical intersection** built in as a first-class workflow (workflows.md workflow 6 + `/outage`'s escalation guidance), given the domain overlap with the user base.
- **US-default, international-aware:** NERC framing is primary; ENTSO-E and IEC cross-references present where relevant.
