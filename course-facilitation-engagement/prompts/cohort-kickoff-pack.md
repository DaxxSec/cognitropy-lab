# Cohort Kickoff Pack — Pre-Launch Prompt

## Purpose
Use this prompt at T-21 to T-7 to render the full pre-launch communication pack across every required stakeholder lane in one pass. The output is a set of approval-ready drafts; the user then routes them through the named approvers from the RACI.

## Prerequisites
- `/onboard` complete and `context/project.md` populated
- `/map-stakeholders` complete and `outputs/stakeholder-register-{cohort_id}.md` available
- `/draft-cadence` complete and `planning/plan.md` carries the pre-launch rows
- All required template variables filled in `context/project.md`

## Prompt Template

I'm running the kickoff pack for {{cohort_name}}. Pull the pre-launch rows from `planning/plan.md`, render the required templates from `resources/stakeholder-template-library.md`, and produce a single approval-ready draft pack. Use the variables in `context/project.md`.

For each artifact, I want:
1. A rendered draft body (no `{{placeholders}}` remaining)
2. The named approver from the RACI
3. The send window (ISO date + time + zone)
4. A one-sentence rationale tying the artifact to the framework it supports (CoI presence, SDT driver, or Five Moments of Need)

The pack must include — at minimum:

- `pre-sponsor-inform-email` (T-30): scope, dates, what they will receive, any Level 3 commitment
- `pre-learner-invite-email` (T-21): warm pitch, goals, joining details, what to expect
- `pre-manager-inform-email` (T-21): heads-up, expected manager role
- `pre-learner-invite-calendar` (T-14): rich calendar invite with prework + welcome Loom
- `pre-manager-nudge-email` (T-14): three 1:1 prompts the manager can use to brief their report
- `pre-learner-remind-email` (T-7): pre-work due, joining details, prep checklist
- `pre-admin-inform-email` (T-3): final cohort manifest + access provisioning
- `pre-learner-remind-email` (T-1): tomorrow's joining details, time zone, what to bring
- `pre-facilitator-dossier-internal` (T-1): cohort overview, accessibility notes, named at-risk-on-arrival learners

Apply checks before output:
- Plain language at Grade 8 for learner copy
- Time-zone-aware send windows; no learner gets a send outside 08:00–20:00 local
- Each artifact has exactly one ask
- UDL: every learner artifact pairs text with at least one alternate representation (Loom, audio, or rich-content calendar)
- One graceful-exit line on every nudge

## Expected Output

A markdown pack at `outputs/{cohort_id}/kickoff-pack-{YYYY-MM-DD}.md` with sections:

```
## Approval-ready pack — {{cohort_name}} kickoff
- Generated: {{generation_iso}}
- Generated by: {{user}}
- RACI source: {{path_to_register}}

## 1. pre-sponsor-inform-email (T-30)
Approver: {{name}}
Send window: {{iso}} — {{tz}}
Framework anchor: Kirkpatrick L4 commitment
Rendered body:
[full body]

## 2. pre-learner-invite-email (T-21)
...

(continue for each artifact)
```

A summary block at the top: count by stakeholder, count by channel, count by approver, and any `<<NEW>>` template gaps surfaced during render.

## Decision Points
- If a manager email lacks a learner-facing parallel, surface the gap before rendering.
- If a sponsor was set Definitive but no chief-of-staff approver was named, halt the pack and ask the user to confirm the approval lane.
- If any `<<NEW>>` template appears in the cadence, render the rest of the pack and surface the missing template name with a one-paragraph spec for the user to author.
