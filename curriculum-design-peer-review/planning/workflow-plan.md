# Curriculum Design Workflow Plan

## Recommended Workflow Order

1. **Onboard** (`/onboard`) — Gather context about subject, audience, standards
2. **Design** (`/design-unit`) — Build a unit using backward design
3. **Review** (`/peer-review`) — Run structured peer review on the unit
4. **Align** (`/align-standards`) — Verify standards coverage
5. **Assess** (`/rubric-gen`) — Generate rubrics for performance tasks
6. **Iterate** — Revise based on review feedback, re-review as needed

## Decision Points

- **New curriculum?** → Start with `/design-unit`
- **Existing materials?** → Start with `/peer-review`
- **Standards audit needed?** → Start with `/align-standards`
- **Just need a rubric?** → Jump to `/rubric-gen`

## Quality Gates

Before marking a unit complete, verify:
1. All objectives use measurable Bloom's verbs
2. Assessment-objective alignment is 100%
3. DOK Level 3+ objectives make up at least 30% of the total
4. UDL principles are addressed
5. Peer review report shows no critical issues remaining
