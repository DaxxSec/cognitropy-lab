# Design Unit

Design a complete curriculum unit using backward design principles.

## Required Input
Ask the user for:
1. **Topic/Theme** — What is the unit about?
2. **Duration** — How many class sessions or weeks?
3. **Key standards** — Which standards should this unit address?
4. **Prior knowledge** — What do learners already know?

## Process

### Stage 1: Desired Results
- Draft 3-5 enduring understandings (big ideas learners should retain)
- Write 2-3 essential questions (open-ended, thought-provoking)
- Write 4-8 learning objectives using Bloom's Revised Taxonomy verbs
- Format each objective: "Students will be able to [verb] [what] [under what conditions]"
- Map objectives to standards using the alignment workflow

Present objectives to user for approval before proceeding.

### Stage 2: Assessment Evidence
- Design one authentic performance task per enduring understanding
- Create formative assessment checkpoints for each objective
- Ensure every objective has at least one assessment measuring it
- Generate rubric for the performance task using rubric workflow

### Stage 3: Learning Plan
- Sequence daily/session activities using WHERE TO framework
- Include varied modalities (lecture, discussion, hands-on, digital, collaborative)
- Build scaffolding from lower Bloom's levels to higher
- Add estimated time per activity
- Note UDL accommodations for each major activity

## Output
Save the complete unit plan to `outputs/unit-[topic]-v1.md`
Save the rubric to `outputs/rubric-[topic]-performance-task.md`
Log the design session to `work-log/`
Suggest running `/peer-review` on the completed unit.
