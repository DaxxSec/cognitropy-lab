# Getting Started

## Quick Start

1. Run `/onboard` to set up your teaching context
2. Use `/design-unit` to create your first unit plan
3. Run `/peer-review` to get structured feedback
4. Iterate until your review shows no critical issues

## Tips for Best Results

- Be specific about your learner audience during onboarding
- Provide your actual standards framework so alignment is accurate
- Start with one unit before designing an entire course
- Use the peer review checklist in `resources/` even for quick self-checks
- Save all iterations — version numbers help track your design evolution
