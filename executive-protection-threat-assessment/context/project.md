# Project Context

> Populated by `/onboard`. The fields below are the canonical schema; `/onboard` will fill them through interview.

## Principal

- **Identifier (codename):** _e.g. PRINCIPAL-ECHO; never the principal's real name in this file._
- **Public profile sensitivity:** _low / moderate / high / extreme._
- **Industry / role context:** _what makes the principal a target — sector, public statements, ongoing litigation, geopolitical exposure._
- **Family / accompanying parties to consider:** _ages, mobility, separate threat profile yes/no._
- **Known prior incidents (last 24 months):** _date / type / outcome / case-number reference._
- **Standing threats (open):** _credible threats currently believed live; reference incident IDs._

## Engagement / Itinerary in scope

- **Engagement window:** _start / end ISO date and timezone._
- **Geography:** _country, region, city, neighbourhood; geopolitical permissiveness rating._
- **Movement profile:** _static / single-leg motorcade / multi-leg motorcade / mixed._
- **Public exposure:** _closed event / semi-public / fully public / press-attended._
- **Local liaison:** _named host-country protective service contact, if any; coordination expectations._

## Mission objectives

- **Primary mission of the principal:** _why they are travelling; what success looks like for them._
- **Detail's mission alignment:** _which detail-level objectives must serve principal's mission (e.g. low-visibility profile vs. visible deterrence)._
- **Non-negotiable mission constraints:** _meetings the principal will not skip; deadlines that anchor scheduling._

## Threat environment summary (snapshot at onboarding)

- **Macro threat picture:** _country-level political stability, recent unrest, terrorism baseline._
- **Sector-specific threat picture:** _activism / commercial / criminal threats specific to principal's sector._
- **Known interest by named actors:** _named actors / groups with demonstrated interest, evidence-graded._
- **Capability inventory of likely adversaries:** _what classes of vehicle, weapon, surveillance capability they have shown._
- **Recent incident base-rate (last 12 mo):** _attacks / surveillance / hostile approaches in this geography or against similar principals._

## Authorities & scope

- **Lawful basis for protection:** _contractual, in-house, statutory; jurisdiction._
- **Use-of-force authority of the detail:** _e.g. private contractor with citizen-arrest authority only; sworn officer; armed authorisation status._
- **Vehicle authority:** _civilian motorcade only / police escort / authorised lane usage / emergency-vehicle privileges._
- **Information-handling authority:** _what classifications the detail can hold; data-protection regime applicable._

## Known constraints (mission, not legal — see `constraints.md` for legal)

- **Time-of-day windows the principal will not move outside of**
- **Any vehicles or routes the principal refuses on personal grounds**
- **Any media / photography rules the principal sets**
